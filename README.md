# Documentação Técnica de Engenharia: Core Framework (V3) Este documento é a especificação oficial e o roteiro de desenvolvimento para o projeto base multi-tenant. Ele consolida as decisões de arquitetura, segurança e produto para garantir um sistema ágil, escalável e inviolável. --- ## 1. Visão Geral do Produto O **Core Framework** é uma infraestrutura de microserviços em Python projetada para servir como base para múltiplos sistemas futuros. * **Objetivo:** Atender até 100 empresas (tenants) com uma única base de código, garantindo isolamento total de dados e segurança de nível corporativo. * **Tecnologias:** Python (Backend), HTML5, CSS3, Bootstrap 5 (Frontend). --- ## 2. Arquitetura Multi-Tenancy e Escala Para equilibrar custo de manutenção e performance, adotamos o modelo **Híbrido**: * **Código-Fonte Único:** Todas as empresas utilizam a mesma versão do software. * **Bancos de Dados Isolados:** Cada CNPJ possui sua própria instância ou esquema de banco de dados, garantindo privacidade e soberania de dados. * **Componentização Granular:** O sistema é dividido em arquivos únicos para cada função (`1 Arquivo = 1 Função`), permitindo manutenção isolada e escalabilidade elástica. --- ## 3. Camada de Segurança "Triple Lock" (Tranca Tripla) Nenhuma função do sistema é executada sem a validação simultânea de três chaves de identificação: | Chave | Tipo | Descrição | | :--- | :--- | :--- | | **ID Único (UUID)** | Transação | Identificador único gerado por processo para rastreabilidade e log. | | **CNPJ** | Tenant | Identificador da empresa para direcionamento do banco de dados. | | **CPF** | Operador | Identificador do usuário responsável pela ação para fins de auditoria. | --- ## 4. Engenharia de Integridade de Código Para evitar alterações não autorizadas ou "tampering" (apagamento de travas), implementamos a **Assinatura Digital de Componentes**: ### A. Fluxo de Validação 1. **Assinatura Offline (Chave Privada):** O desenvolvedor assina o hash de cada arquivo de função fora do ambiente de produção. 2. **Verificação em Runtime (Chave Pública):** Antes de executar qualquer função, um orquestrador seguro (Secure Loader) valida se o conteúdo do arquivo bate com a assinatura digital. 3. **Bloqueio Automático:** Caso uma linha de código seja alterada ou a assinatura removida, o processamento para imediatamente e um alerta de segurança é disparado. ### B. Proteção contra Alteração * **Compilação de Segurança:** O componente de validação é compilado em binário (Cython/.so) para evitar leitura ou modificação do código de proteção. * **Log de Auditoria (Caixa Preta):** Todas as tentativas de execução sem chaves válidas são gravadas em um banco de dados centralizado e imutável. --- ## 5. Rotinas Comuns (Microserviços Base) Funcionalidades que serão herdadas por todos os programas futuros: 1. **IAM (Gestão de Identidade):** Cadastro e login com hash de senha (Bcrypt). 2. **Gateway de Conexão:** Orquestrador que seleciona o banco de dados via CNPJ. 3. **Auditoria Central:** Registro de todas as transações baseadas no Triple Lock. 4. **Tematização Dinâmica:** Injeção de cores e logos via CSS/Bootstrap baseado no perfil da empresa. --- ## 6.
